# RISK_REGISTER.md

| Риск                                                       | Признак/доказательство                                                              | Вероятность | Влияние   | Меры снижения                                                        |
|------------------------------------------------------------|------------------------------------------------------------------------------------|-------------|-----------|-----------------------------------------------------------------------|
| Потеря данных БД при рестарте контейнера                   | PaaS Render использует эфемерное хранилище для SQLite. База обнуляется при деплое. | 100%        | Высокое   | Заменить SQLite на PostgreSQL (внешний сервис).                       |
| Сбой API Telegram (таймаут)                                | При отправке уведомления запрос может зависнуть на 5 сек, блокируя ответ клиента. | Средняя     | Среднее   | Вынести отправку в фоновые задачи (Celery, Redis Queue).              |
| Нехватка RAM на бесплатном тарифе Render                   | Приложение может превысить 512 MB при одновременных запросах.                     | Низкая      | Высокое   | Оптимизировать запросы ORM, добавить кэширование.                    |
| Утечка SECRET_KEY через переменные окружения               | Хранение ключа в открытом виде в панели Render.                                    | Низкая      | Высокое   | Регулярная ротация ключей, использование секретов от Render.          |
| Отсутствие валидации типов (количество передаётся строкой) | Пользователь может отправить "abc" в поле количества – вызовет 500 ошибку.        | Средняя     | Среднее   | Добавить валидацию на бэкенде (Flask-WTF или try/except).             |
| Логи не содержат контекста для отладки                     | В логах только INFO и ERROR без ID пользователя, IP, endpoint.                   | Высокая     | Среднее   | Настроить структурированное логирование (JSON, доп. поля).            |
