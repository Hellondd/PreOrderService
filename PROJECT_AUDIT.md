# Аудит проекта и рефакторинг кода (Этап 1)

## 1. Цель аудита
Анализ текущего состояния проекта `PreOrder_service`, выявление уязвимостей, архитектурных и синтаксических ошибок, приведение кода к стандарту PEP8 перед подготовкой к релизу.

## 2. Инструменты аудита
* **Линтер:** `flake8` (проверка синтаксиса и стиля)
* **Форматтер:** `black` (автоформатирование кода)
* **Анализ безопасности:** ручная инспекция кода на наличие хардкода секретов.

## 3. Выявленные проблемы и решения

| Модуль / Файл | Проблема (ДО) | Решение (ПОСЛЕ) | Статус |
| :--- | :--- | :--- | :--- |
| `identity.py` | Критическая ошибка: дублирование метода `authenticate`. | Лишний метод удален, логика отладки и проверки объединена. Добавлен декоратор `@staticmethod`. | Исправлено |
| `notifications.py` | Уязвимость (CWE-798): токен Telegram-бота жестко зашит в код. | Секретный ключ вынесен в `.env`. Для чтения используется `os.getenv('TELEGRAM_TOKEN')`. | Исправлено |
| `orders.py` | Нарушения PEP8 (E302, W293, E501). Длина строк превышала 120 символов. | Код отформатирован. Длинные запросы ORM разбиты на несколько строк. | Исправлено |
| Архитектура БД | Отсутствие изоляции при тестировании. | Тестовая БД вынесена в фикстуры `pytest` с использованием отдельного URI. | Исправлено |

## 4. Результат
* Весь проект отформатирован в соответствии со стандартами разработки на Python.
* Линтер `flake8` отрабатывает без предупреждений (`exit-zero`).
* Секретные данные изолированы от исходного кода. Проект готов к безопасному развертыванию.
